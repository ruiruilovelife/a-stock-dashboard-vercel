# A股跟踪仪表盘系统级重构验收报告

更新时间：2026-07-11

## 1. 新旧架构

旧架构由单一更新脚本内的多组名单、规则和页面固定文案分别生成结论，行业标签、目标市值和动作可能互相矛盾。

新架构统一为：行情/财务/公告/新闻/海外数据 → 时效与质量校验 → 行业识别 → 业务结构和转型评分 → 行业专用估值/SOTP → 公司研究快照 → 事件、资金、海外信号 → 三个选股模型和持仓动作 → 页面。

## 2. 数据流与调用关系

- `unified-research-engine.mjs` 是公司、业务、行业、估值和结论的唯一底层入口。
- `decision-engine.mjs` 读取统一公司研究快照，生成首席观点、持仓动作、资金结构、事件中心、海外传导和数据健康。
- `industry-map-engine.mjs` 读取同一公司业务标签和事件中心，生成独立产业地图。
- 五倍股、估值质量、低估值和强弹性只读取统一估值结果，不再保存独立目标市值。

## 4. 已迁移模型

- 五倍股：统一中性估值、排名有效性和增长贡献拆解。
- 估值质量：统一行业方法、三情景和价值陷阱检测。
- 低估值：仅比较有效的行业专用保守/中性估值。
- 强弹性：基本面空间读取统一引擎；技术和资金只判断启动时机。
- 持仓建议：当前价格、目标价和动作均来自同一研究快照。

## 5. 行业参数完整字段

一级/二级行业、主方法、辅助方法、禁止方法、三情景参数、历史中枢、同行中枢、上下限、周期属性、成长属性、关键指标、参数来源、更新时间、版本、置信度和最大有效期均在配置文件维护。

## 6. 主要行业估值方法

- 保险：P/EV、PB，正常化PE辅助；禁止普通PS。
- 银行：PB，正常化PE辅助。
- 券商：PB和周期中枢PE。
- 周期资源/化工：中周期PE、PB、EV/EBITDA。
- 半导体/高端制造：远期PE，PS/PEG辅助。
- 软件平台：盈利时远期PE/PEG，未盈利时PS。
- 创新药：风险调整管线估值/DCF；无管线数据时失效。
- 消费：PE/PEG/DCF。
- 公用事业：DCF/股息率，PE/PB辅助。
- 地产：NAV/PB。

## 7. 动态业务识别

保留法定行业、原主营、核心收入、核心利润、新增长业务、市场定价逻辑、商业化阶段、新旧业务权重、调整原因和复核时间。关键词、框架协议、少量参股和股价炒作不能触发估值迁移。

## 8. 转型评分

0—100分，由新业务收入、利润、订单、增长、毛利率、研发、资本开支、客户质量、商业化和数据可信度等确定性字段计算。0—20不调整，21—40少量期权，41—60 SOTP，61—80提高新赛道权重，81—100允许主逻辑切换。

## 9. SOTP

仅在存在至少两个可验证分部且转型评分达到41分时启用：传统业务价值 + 新业务价值 + 其他资产 - 净负债 - 风险折价。未产生收入的新业务不得获得成熟赛道倍数。

## 10. 估值逻辑升降级

订单、收入、利润、客户和商业化成熟度提高时逐级提升新业务权重；订单取消、收入下降、商业化失败或数据可信度下降时自动降低权重并触发重算。

## 11. 新华保险15,719亿元错误来源

旧通用公式把保险公司当普通成长公司，同时使用利润和收入增长路径：当前市值约1,964.7亿元，PE约5.3、PS约1.3；收入CAGR 48.5%、利润CAGR 104.1%被分别封顶到45%和60%，再外推增长并套18倍PE、2倍PS。两条路径约为27,331亿元和9,215亿元，中位数约18,273亿元，最终被“当前市值×8”上限截成约15,718亿元。根因是行业与方法错配及重复增长外推，不是手工目标值。

## 12. 新华保险修改后

改用保险PB交叉估值；测试情景约989/1,570/1,944亿元。真实数据运行时若内含价值缺失，仅保留PB参考并禁止参与空间排名。东方财富异常负股本被识别后，目标价改用当前市值÷股价反推股本，避免目标价为0。

## 13. 其他异常扫描

系统统一拦截金融使用PS、周期利润高点、未盈利公司使用PE、未知行业、股本异常、单位缺失、目标价不一致、数据过期和新业务过度估值。此前博俊科技、沪光股份等旧公式触发“当前市值×8”上限的结果不再沿用。

## 14. 独立产业地图

已从五倍股页拆出，覆盖12个一级产业、78个二级赛道。每条赛道包括定义、阶段、3—5年空间、景气、政策/技术驱动、商业化、上中下游、公司关系、估值、风险、事件、来源和时间。

## 15. 首席观点

只保留市场阶段、3—5条核心结论、资金迁移与行业状态、持仓行动摘要、未来1—5日观察和数据健康。删除风格帽和重复候选列表。

## 16. 持仓建议

动作限定为继续持有、回调/突破加仓、持有观察、减仓、清仓、暂不操作、等待财报或技术确认；同时展示优先级、三情景距离、趋势、量能、板块、公告、业务兑现、组合角色、触发条件、有效期和置信度。

## 17. 资金情绪

展示五个核心指数、市场宽度、涨跌停、涨跌中位数、强弱数量、情绪周期和资金迁移矩阵。公开源未稳定提供的ETF申赎、融资、大单、炸板率等明确列为未接入，不使用推测值。

## 18. 投研雷达

按公司、日期和标准化标题聚合去重，分类宏观、政策、产业、技术、基本面、订单、价格和风险。每条事件含来源、可信度、方向、短中期影响、传导路径、是否调整盈利/估值/评分、有效期和失效条件。

## 19. 财报雷达

聚合财报、预告、快报、订单、减持和监管事件；利润必须拆分主营、销量、价格、毛利率、费用、投资收益和非经常损益。缺少现金流、应收、存货和合同负债时标记待核验。

## 20. 海外映射

按海外市场/公司/政策 → 全球产业链 → 产品价格/订单/资本开支 → A股行业 → 真实业务敞口 → 收入/利润/估值/情绪构建，不以单日涨跌直接推荐股票。

## 21. 全球政策评分

出口限制影响国产替代与风险折价；日本设备政策影响设备材料；韩国存储周期影响存储/HBM；利率和汇率影响成长估值与出口利润。信号先进入事件层，再影响行业、公司评分和动作。

## 22. 时效与失效

显示行情、财务、公告、新闻、海外和模型时间。股价变化超过5%、财报/重大事件、新业务比例、转型分、行业参数、盈利预测或景气变化，以及接口失败均触发重算或失效。

## 23. 页面调整

导航改为首席观点、操作建议、资金与情绪、投研雷达、财报与公告、全球与海外映射、产业地图、强弹性、估值质量、五倍股、低估值、策略纪律。取消我的跟踪池独立入口，历史买卖记录仍保留在数据层。

## 24. 固定文案

首席观点、操作建议和策略纪律的新页面不再显示旧仓位数字、旧股票动作、风格帽和固定方向结论；页面从统一对象实时渲染。
